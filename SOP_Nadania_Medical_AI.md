# คู่มือมาตรฐานการปฏิบัติงาน (SOP)
**ระบบ:** Nadania Medical AI Auto-Reels Pipeline (ระบบผลิตวิดีโออัตโนมัติ)
**เวอร์ชัน:** 2.0 (เชื่อมต่อระบบ Google Ecosystem / Imagen 4.0)
**วัตถุประสงค์:** สร้างวิดีโอสั้นอัตโนมัติ 100% โดยที่มนุษย์ไม่ต้องแตะต้อง เพื่อดึงดูดผู้คนเข้าชมเพจ (Organic Traffic) แนะนำผู้มุ่งหวังเข้าสู่ระบบแบบประเมินสุขภาพ และสร้างภาพลักษณ์ผู้เชี่ยวชาญด้าน Medical AI & การฟื้นฟูระดับเซลล์

---

## 1. ภาพรวมของระบบ (System Overview)

การทำงานทั้งหมดจะวิ่งอยู่บน **GitHub Actions** (เซิร์ฟเวอร์หลังบ้าน) และถูกบริหารผ่าน **Web Dashboard** (หน้าปัดควบคุม)

- **สมองหลักคิดคอนเทนต์:** Google Gemini 2.5 (ทำหน้าที่เขียนสคริปต์พูด และแคปชั่น)
- **เครื่องมือวาดภาพ:** Google Imagen 4.0 (วาดรูปประกอบที่มีความสมจริงระดับภาพถ่าย หรือ Photorealistic)
- **ระบบเสียง:** Google Translate TTS (พากย์เสียงภาษาอังกฤษ โทนทางการแพทย์)
- **ผู้จัดการตัดต่อ:** FFmpeg (ทำหน้าที่เอา เสียง + รูปแบบมาประกอบร่างกันเป็นวิดีโอ .mp4 ความยาว 15 วินาที)
- **พนักงานส่งของ:** Facebook Graph API (คอยอัปโหลดวิดีโอขึ้นสู่เพจ Facebook โดยไม่ต้องเปิดแอป)
- **โกดังเก็บข่าว:** Aiven MySQL Database (ฐานข้อมูลสำหรับตุนข่าว ไม่ให้ AI ทำคลิปซ้ำหัวข้อเดิม)

## 2. กุญแจสำคัญและตัวแปรระบบ (API Keys & Secrets)

ระบบนี้จะพังทันทีถ้ากุญแจ (API Key) หมดอายุหรือหายไป ดังนั้นต้องตรวจสอบให้แน่ใจว่าติดตั้งคีย์เหล่านี้ไว้ครบทั้งในไฟล์ `.env.local` (ฝั่ง Dashboard) และ **GitHub Actions Secrets** (ฝั่งเซิร์ฟเวอร์ยิงอัตโนมัติ)

*   `GEMINI_API_KEY`: คีย์ของ Google AI Studio (ใช้สำหรับให้ AI คิดคำ และวาดภาพ Imagen 4.0) **สำคัญที่สุด**
*   `FB_PAGE_ACCESS_TOKEN` & `FB_PAGE_ID`: กุญแจของ Meta สำหรับอนุญาตให้ระบบโพสต์วิดีโอลงเพจแทนมนุษย์ได้
*   `MYSQL_HOST`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`, `MYSQL_PORT`: รหัสเข้าถึงฐานข้อมูล Aiven
*   `GITHUB_PAT`: รหัสส่วนบุคคล (Personal Access Token) ตั้งค่าไว้ใน Vercel เท่านั้น เพื่อให้ปุ่มจากหน้าเว็บสามารถสั่งปลุก GitHub ให้ตื่นมาแคสต์งานได้ทันที

> [!WARNING]
> หากจู่ๆ ระบบหยุดโพสต์วิดีโอ ให้พุ่งเป้าไปที่ "กุญแจหมดอายุ" เป็นอันดับแรก โดยเฉพาะ `GEMINI_API_KEY` หากออกกุญแจใหม่ ต้องนำไปอัปเดตทั้งใน Vercel และ GitHub Secrets ทันที

## 3. กลไกการทำงานประจำวัน (Autopilot Mode)

ระบบนี้ถูกดีไซน์มาให้ทำงานแบบอัตโนมัติ โดยเจ้าของเพจกินเตี่ยวตีพุงรอผลลัพธ์ได้เลย

1. **จังหวะชีพจร:** GitHub Actions (`medical_video_cron.yml`) จะตื่นขึ้นมาทำงาน **วันละ 6 รอบ**
2. **ตารางเวลาหลบรถติด:** เพื่อเลี่ยงอาการเซิร์ฟเวอร์ GitHub อืด ระบบจะถูกตั้งเวลาให้ทำงานในนาทีที่ลงท้ายแบบแปลกๆ (นาฬิกา UTC): 
    *   `00:07`, `04:17`, `08:27`, `12:37`, `16:47`, `20:57`
3. **ตรรกะการหยิบข่าว:**
    *   เข้าไปส่องในคิวตาราง `health_reels_queue`
    *   ดึงหัวข้อที่ค้างอยู่ (Pending) บนสุดมาทำคลิป
    *   **การตุนสต๊อกอัตโนมัติ (Self-Replenishing):** ถ้าปรากฏว่าคิวข่าว "ว่างเปล่า" ระบบจะวิ่งออกไปที่ Google News ดึงบทความเรื่องสุขภาพ/เทคโนโลยีล่าสุดมาตุนไว้ 10-20 หัวข้อทันที จากนั้นค่อยเริ่มงาน
4. **ตัดต่อและเผยแพร่:** เขียนบท > พากย์เสียง > วาดภาพ > สั่ง FFmpeg ตัดต่อคลิป > ส่ายตูดเดินนำไปโพสต์ขึ้น Facebook
5. **ดักตีกินลูกค้า (CTA):** เมื่อโพสต์วิดีโอเสร็จ ระบบจะเอานามสกุลของตัวเองเข้าไปคอมเมนต์ใต้โพสต์นั้นทันทีด้วยประโยคเชิญชวน:
    * *"🩺 อยากรู้สภาวะการทำงานของเซลล์ร่างกายคุณไหม? ลองใช้แบบประเมิน Medical AI ระดับคลินิกของเรา ฟรี! ได้ตั้งแต่วันนี้ที่: https://nadaniadigitalllc.com/wellness"*

## 4. การควบม้าเองด้วยมือ (Manual Dashboard Trigger)

หากมีข่าวฉุกเฉิน หรือบอสต้องการเร่งโพสต์โดยไม่อยากรออีก 4 ชั่วโมงให้ถึงรอบถัดไป:

1.  เข้าไปที่หน้าคควบคุม: `https://nadaniadigitalllc.com/admin/reels-queue`
2.  สังเกตแบนเนอร์สีม่วง **Next Automated Post Pipeline** ซึ่งจะคำนวณและคาดการณ์เวลาในการรันรอบถัดไปเป็นเวลาท้องถิ่น
3.  กดตะปุ่ม **⚡️ Force Post Now** (สีขาว)
4.  หน้าเว็บจะส่งสัญญาณปืนใหญ่ (GitHub PAT) ไปกระชากคอให้ GitHub ตื่นขึ้นมารันคำสั่งทำวิดีโอเดี๋ยวนั้นทันที!
5.  รอประมาณ 1-2 นาที แวะไปดูหน้าเพจ Facebook เพื่อดูวิดีโอที่เพิ่งโพสต์สดๆ ร้อนๆ ได้เลย

## 5. การเจาะตลาดด้วยแอดโฆษณา (Paid Advertising Workflow)

หากต้องการเร่งยอดคนเข้าเว็บโหดๆ ให้ใช้ฟีเจอร์ A/B Testing สำหรับทำภาพนิ่งไว้กด "Boost Post" (ยิงแอด)

- **เป้าหมาย:** ทำแอด 2 สไตล์ (เช่น สาย "ชะลอวัย/Anti-Aging" ชนกับคู่สาย "คนบ้างาน/ออฟฟิศซินโดรม") เพื่อดูว่ากลุ่มไหนคลิกมากกว่า และคุ้มค่าโฆษณาที่สุด
- **วิธีรัน:** เรียกใช้คำสั่งใน Terminal ด้วย `node web-dashboard/post_fb_ads.js`
- **สิ่งที่ได้:** ระบบจะคำนวณเนื้อหา สร้างก๊อปปี้โฆษณายาวรวดร้าวถึงทรวง แล้วเจเนอเรตภาพนิ่ง (Photo) พร้อมโพสต์ลงหน้าเพจ (เหมาะนำจุดนี้ไปกดยิง Ads ใน Meta Business Suite ต่อ)

## 6. วิธีเปลี่ยนพฤติกรรมและนิสัยของบอท AI (Modifying Prompts)

ถ้าบอสเบื่อสไตล์ของวิดีโอ และอยากแก้ไขมูดโทน สามารถทำได้โดยเข้าไปที่ไฟล์: `web-dashboard/generate_health_reel.js`

*   **เปลี่ยนนิสัยคำพูด:** ตรงจุดที่เขียนว่า `const prompt = ...` คือหัวใจหลัก บอสสามารถเข้าไปเปลี่ยนกฎเกณฑ์ของมันได้เลย เช่น บังคับให้เขียนเน้นแต่เรื่อง "น้ำหนัก" หรือ "เบาหวาน"
*   **เคล็ดลับท้ายคลิป:** ปัจจุบันผมสั่งให้บอท "ต้องหยอดเทคนิคสุขภาพสั้นๆ 2-3 ข้อ ไว้แคปชั่นตอนจบเสมอ" บอสสามารถเข้าไปลบคำสั่งนี้ หรือปรับแต่งได้ตามใจ
*   **โทนภาพ:** หากเบื่อการถ่ายภาพแนวสมจริง ภาพเชิงการแพทย์ สามารถลบคำว่า `Create a highly detailed, cinematic, photorealistic...` ทิ้ง และป้อนศิลปะแนว การ์ตูน, ภาพสีน้ำ, ภาพล้ำยุค ลงไปแทนได้ครับ

## 7. ขอบเขตการแก้ปัญหาเบื้องต้น (Troubleshooting Guide)

| อาการเสียที่พบเจอบ่อย | สาเหตุ | วิธีแก้ปัญหากำปั้นทุบดิน |
| :--- | :--- | :--- |
| **รอเป็นวันแล้ว วิดีโอก็ไม่โพสต์อัตโนมัติ** | เซิร์ฟเวอร์ GitHub ช้าเกินรับไหว / หรือ `GEMINI_API_KEY` หมดสถานะการใช้งาน | เข้าไปดูในแท็บ Action บนหน้า Github ว่ามันโชว์กากบาทสีแดงหรือไม่ ถ้าใช่ ให้ไปเอาคีย์ตัวใหม่มาเติมใน GitHub Secrets |
